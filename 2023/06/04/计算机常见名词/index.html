计算机常见名词

发表于 2023-06-04 分类于 计算机小白需要知道的常识
本文字数： 721 阅读时长 ≈ 3 分钟

  • 计算机小白可能见到的各种计算机名词的介绍

    不定期持续更新 ~

常识

  1. OOP和POP

    OOP：Object Oriented Programming，面向对象编程

    POP：Procedure Oriented Programming，面向过程编程

  2. bin和src

    bin：binary，二进制。bin文件夹里面存放的一般是可执行的二进制文件

    src：source，源。src文件夹里面存放的是源文件，对 Java 来说就是后缀为.java的文件

  3. DOS：Disk Operating System，磁盘操作系统

  4. OS：Operating System，操作系统

  5. argument 和 parameter

    parameter 是函数定义中的参数，形式参数

    argument 是函数调用时的参数，实际参数

  6. URL：Uniform Resource Locator，统一资源定位器

  7. MinGW：Minimalist GNU for Windows 的缩写，是一个可自由使用和自由发布的Windows特定头文件和使用GNU工具集导入库的集合，允许你在GNU/Linux 和 Windows 平台生成本地的Windows程序而不需要第三方C运行时（C Runtime）库。

开发

  1. API和SDK

    API：Application Programming Interface，应用程序接口，用来提供应用程序与开发人员基于某软件或硬件得以访问的一组例程，而又无需访问源码，或理解内部工作机制的细节。

    SDK：Software Development Kit，软件开发工具包，软件开发工具包一般都是一些软件工程师为特定的软件包、软件框架、硬件平台、操作系统等建立应用软件时的开发工具的集合。

  2. 幂等性：多次调用方法或者接口不会改变业务状态，可以保证重复调用的结果和单次调用的结果一致。

  3. RPC：Remote Procedure Call，远程函数调用。假设有两台服务器 A，B，一个应用部署在 A 服务器上，想要调用 B 服务器上应用提供的函数/方法，由于不在一个内存空间，不能直接调用，需要通过网络来表达调用的语义和传达调用的数据。

  4. IDL：Interface description language，接口描述语言，通过一种中立的方式来描述接口，使得在不同平台上运行的对象和用不同语言编写的程序可以相互通信交流；比如，一个组件用C++写成，另一个组件用Java写成。

  5. 动态编译（dynamic compilation）指的是“在运行时进行编译”；与之相对的是事前编译（ahead-of-time compilation，简称AOT），也叫静态编译（static compilation）。

  6. JIT：just-in-time compilation，即时编译，狭义来说是当某段代码即将第一次被执行时进行编译，因而叫“即时编译”。JIT编译是动态编译的一种特例。JIT编译一词后来被泛化，时常与动态编译等价；但要注意广义与狭义的JIT编译所指的区别。
